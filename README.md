# MarvelMind — MCU Film Recommendation System

A hybrid machine learning recommendation engine for Marvel Cinematic Universe films, featuring a TF-IDF plot similarity model, a NetworkX cast graph, and a cinematic frontend with live TMDB poster data.

## How It Works

MarvelMind uses a **hybrid recommendation model** combining two signals:

### 1. Plot DNA — TF-IDF Cosine Similarity
Each film's plot is vectorised using `TfidfVectorizer`. Cosine similarity scores measure how narratively close two films are, capturing shared themes, tone, and story structure.

### 2. Cast Graph — NetworkX
Every actor is a node. Every shared film creates a weighted edge between co-starring actors. Cast overlap between two films is calculated from this graph and normalised into a score.

### 3. Hybrid Fusion
Both scores are blended using a tuneable **alpha weight**:

```
final_score = (alpha × plot_score) + ((1 - alpha) × cast_score)
```

The frontend's tone selector maps to different alpha values:

| Tone | Alpha | Behaviour |
|------|-------|-----------|
| Balanced | 0.6 | 60% plot, 40% cast |
| Cinematic | 0.75 | Plot-heavy |
| Action-heavy | 0.4 | Cast-heavy |
| Story-first | 0.8 | Very plot-heavy |
| Hidden Gems | 0.55 | Slight plot lean |

## API Endpoints

The Flask backend runs on `http://localhost:5000`.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/films` | Returns all films from the dataset |
| `POST` | `/recommend/movie` | Hybrid film recommendations |
| `POST` | `/recommend/character` | Actor filmography + graph co-stars |
| `GET` | `/api/films/<title>` | Single film detail |
| `GET` | `/health` | Backend status and model stats |

### POST `/recommend/movie`
```json
{
  "movie": "Iron Man",
  "alpha": 0.6,
  "top_n": 5
}
```

**Response:**
```json
[
  {
    "title": "Iron Man 2",
    "year": "2010",
    "score": 87,
    "explanation": "Shares cast with Iron Man: Robert Downey Jr., Gwyneth Paltrow",
    "genres": ["Action", "Sci-Fi"],
    "cast": ["Robert Downey Jr.", "..."]
  }
]
```

### POST `/recommend/character`
```json
{
  "character": "Robert Downey Jr.",
  "top_n": 5
}
```

**Response:**
```json
{
  "actor": "Robert Downey Jr.",
  "film_count": 7,
  "phase_count": 3,
  "films": [...],
  "costars": ["Chris Evans", "Scarlett Johansson", "..."]
}
```

## Setup & Installation

### Prerequisites
- Python 3.8+
- pip

### 1. Clone the repository
```bash
git clone https://github.com/your-username/marvelmind.git
cd marvelmind
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Prepare the dataset
If you have the raw CSV, run the preprocessing pipeline first:
```bash
cd src
python data_preprocessing.py
```
This cleans the data, fixes titles, updates statuses, and saves the processed CSV to `data/processed/final_marvel_dataset.csv`.

### 4. Run the Flask backend
```bash
cd src
python app.py
```
You should see:
```
[MarvelMind] Building TF-IDF similarity matrix...
[MarvelMind] Building cast graph...
[MarvelMind] Ready — 40 films loaded, 33 released.
 * Running on http://127.0.0.1:5000
```

### 5. Serve the frontend
Open a second terminal:
```bash
cd src
python -m http.server 8080
```
Then open **http://localhost:8080/index.html** in your browser.

## TMDB Integration (Optional)

The frontend fetches live movie poster images from [The Movie Database API](https://www.themoviedb.org/). The TMDB key is **not included** in the repo — you need to add your own.

1. Create a free account at [themoviedb.org](https://www.themoviedb.org)
2. Go to **Settings → API → Create → Developer**
3. Copy your API Key (v3)
4. Open `src/index.html` and replace the `TMDB_KEY` value near the top of the `<script>` block:

```js
const TMDB_KEY = 'YOUR_TMDB_API_KEY'; // replace this
```

The app works fully without a TMDB key — it loads all films from your Flask dataset and shows placeholder cards instead of posters.

## Dataset

The dataset (`final_marvel_dataset.csv`) contains the following fields:

| Column | Description |
|--------|-------------|
| `title` | Film title |
| `cast` | List of main cast members |
| `genres` | List of genres |
| `plot` | Film plot summary |
| `release_date` | Release date |
| `status` | `released` or `upcoming` |
| `phase` | MCU phase number (1–5) |

The preprocessing pipeline (`data_preprocessing.py`) handles:
- Column renaming and type fixing
- Title corrections (e.g. "Captain Marvel 2" → "The Marvels")
- Status derivation from release date
- Missing film additions (Deadpool & Wolverine, upcoming Phase 5 films)
- Duplicate removal and chronological sorting

## Frontend Features

- **Film Grid** — All MCU films with TMDB posters, ratings, genres, and director info
- **Phase Filter** — Browse films by MCU phase (I–V)
- **Movie Mode** — Select a film, choose a recommendation tone, get hybrid model results with similarity scores
- **Character Mode** — Search by actor name, view full filmography, timeline view, and graph-based co-star recommendations
- **TMDB Enrichment** — Live poster images, cast, runtime, and box office data loaded progressively on startup

---

## Development Notes

- The TF-IDF matrix and cast graph are built **once at startup** and held in memory — subsequent API calls are fast
- Only `released` films are used for the recommendation model; `upcoming` films are shown in the UI but excluded from scoring
- CORS is enabled on all Flask routes so the frontend can call the API from any localhost port
